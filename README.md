# Mini-CRM на Spring Boot

## Опис проєкту

Даний проєкт є лабораторною роботою з розробки REST-застосунку на базі Spring Boot.  
У межах роботи реалізовано клієнт-серверний застосунок mini-CRM для управління менеджерами, клієнтами та історією взаємодій з клієнтами.

Система дозволяє:
- зберігати менеджерів;
- зберігати клієнтів;
- прив’язувати клієнтів до менеджерів;
- зберігати історію взаємодій з клієнтами;
- імпортувати клієнтів із зовнішнього JSON API;
- виконувати базові CRUD-операції через REST API.

## Мета роботи

Метою роботи є розробка повноцінного REST-застосунку мовою Java з використанням Spring Boot, PostgreSQL та Spring Data JPA, який реалізує прикладну бізнес-логіку у вигляді mini-CRM системи.

## Використані технології

- Java 17
- Spring Boot 3.x
- Spring Web
- Spring Data JPA
- PostgreSQL
- Thymeleaf
- Maven
- Validation API

## Структура проєкту

Проєкт організований за багатошаровою архітектурою:

- `controller` — REST-контролери
- `service` — бізнес-логіка
- `repository` — доступ до бази даних
- `entity` — сутності бази даних
- `dto` — DTO-об’єкти для передачі даних
- `templates` — HTML-шаблони
- `resources` — конфігураційні файли

## Основні сутності

У системі реалізовано такі сутності:

### Manager
Зберігає інформацію про менеджера:
- ім’я;
- прізвище;
- email.

### Client
Зберігає інформацію про клієнта:
- повне ім’я;
- email;
- телефон;
- компанія;
- статус;
- менеджер, який відповідає за клієнта.

### Interaction
Зберігає історію взаємодії з клієнтом:
- тип взаємодії;
- опис;
- дата і час;
- клієнт, до якого належить взаємодія.

## REST API

У застосунку реалізовано такі основні endpoint-и:

### Менеджери
- `GET /api/managers`
- `GET /api/managers/{id}`
- `POST /api/managers`
- `PUT /api/managers/{id}`
- `DELETE /api/managers/{id}`

### Клієнти
- `GET /api/clients`
- `GET /api/clients/{id}`
- `POST /api/clients`
- `PUT /api/clients/{id}`
- `DELETE /api/clients/{id}`
- `GET /api/clients/status/{status}`
- `GET /api/clients/manager/{managerId}`
- `GET /api/clients/search?name=...`

### Взаємодії
- `GET /api/interactions`
- `GET /api/interactions/{id}`
- `GET /api/interactions/client/{clientId}`
- `POST /api/interactions`
- `PUT /api/interactions/{id}`
- `DELETE /api/interactions/{id}`

### Імпорт даних
- `POST /api/import/clients/{managerId}`

### Імпорт даних
- `POST /api/import/clients/{managerId}`

## База даних

У проєкті використовується PostgreSQL.  
Створено три основні таблиці:
- `managers`
- `clients`
- `interactions`

Між таблицями реалізовано зв’язки:
- один менеджер може мати багато клієнтів;
- один клієнт може мати багато взаємодій.

## Зовнішній API

Для виконання вимоги щодо отримання даних із зовнішнього джерела реалізовано модуль імпорту клієнтів із публічного JSON API:

- `https://jsonplaceholder.typicode.com/users`

Отримані дані перетворюються у внутрішні об’єкти клієнтів та зберігаються у базі даних.

## Запуск проєкту

Для запуску проєкту необхідно:
1. переконатися, що PostgreSQL запущений;
2. створити базу даних `mini_crm_db`;
3. вказати параметри підключення в `application.properties`;
4. запустити клас `MiniCrmApplication`;
5. відкрити веб-інтерфейс у браузері або використати Postman для перевірки REST endpoint-ів.

Основні сторінки веб-інтерфейсу:
- `/`
- `/clients-page`
- `/managers-page`
